FLUJO TÉCNICO DETALLADO
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CREAR USUARIO:
1. Admin accede a: Menú → SISTEMA → Gestionar Usuarios
2. Hace clic en: "+ Nuevo Usuario"
3. Completa formulario:
   - Nombre (validado: no vacío)
   - Email (validado: único y formato válido)
   - Contraseña (validado: 6+ caracteres)
   - Rol (validado: admin/teacher/parent/student)
4. Haz clic: "✓ Crear Usuario"
5. AdminUsersScreen valida todo
6. Guarda en: localStorage['sistema_usuarios']
7. Mensaje: "✓ Usuario creado correctamente"

USUARIO INTENTA LOGIN:
1. Va a LoginScreen
2. Escribe email y contraseña
3. LoginScreen busca en localStorage['sistema_usuarios']
4. Si encontrado y contraseña = correcta:
   → Crea objeto User
   → Llama onLogin(user)
   → Redirige a App
5. Si no encontrado:
   → Intenta API como fallback
   → Si API falla → "Email o contraseña incorrectos"

APP VALIDA ACCESO:
1. App.tsx recibe User
2. Verifica role en canAccessScreen()
3. Filtra menús según roleAccess
4. Solo muestra pantallas permitidas
5. Si intenta acceder sin permiso → redirige a inicio
