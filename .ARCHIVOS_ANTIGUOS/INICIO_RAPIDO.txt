🎯 5 PASOS PARA DEPLOYAR (30 minutos)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣ CREAR BASE DE DATOS EN TURSO
   ──────────────────────────────
   👉 https://turso.tech
   • Sign up (gratis)
   • Create new database
   • Copia: Connection URL + Auth Token
   ⏱️ 5 minutos


2️⃣ PREPARAR PROYECTO LOCALMENTE
   ──────────────────────────────
   👉 Terminal:
      cd /Users/osmer/Downloads/Abril\ 2026/Sistemita_Nuevo
      cp .env.local.example .env.local

   👉 Editar .env.local con:
      - TURSO_CONNECTION_URL (de Turso)
      - TURSO_AUTH_TOKEN (de Turso)
      - JWT_SECRET (cualquier texto largo)
      - VITE_API_URL (https://tu-proyecto.vercel.app)

   👉 Instalar:
      npm install
   ⏱️ 5 minutos


3️⃣ PROBAR LOCALMENTE
   ──────────────────
   👉 npm run build
   👉 npm run preview
   👉 Abre: http://localhost:4173
   👉 Login: admin@sistemita.edu / admin123
   ⏱️ 5 minutos


4️⃣ DEPLOYAR EN VERCEL
   ───────────────────
   👉 https://vercel.com
   • Sign up con GitHub
   • Import Project
   • Settings → Environment Variables
     (Copia las 5 variables del .env.local)
   • Deploy
   ⏱️ 10 minutos


5️⃣ INICIALIZAR BASE DE DATOS
   ──────────────────────────
   👉 Instalar Turso CLI:
      npm install -g @turso/cli

   👉 Conectar:
      turso auth login
      turso db shell sistemita_db

   👉 Ejecutar SQL (ver DEPLOY_TURSO_VERCEL.md)
   ⏱️ 5 minutos

❌ "Build failed en Vercel"
✅ Solución: npm run build localmente primero

❌ "Database connection failed"
✅ Solución: Verifica TURSO_CONNECTION_URL y TURSO_AUTH_TOKEN

❌ "Token inválido"
✅ Solución: Mismo JWT_SECRET en todas partes

❌ "Email ya existe"
✅ Solución: Usa diferente email para probar
