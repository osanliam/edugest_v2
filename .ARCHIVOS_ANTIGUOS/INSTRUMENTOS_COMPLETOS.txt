1. ✅ RÚBRICA DE EVALUACIÓN
   Archivo: rubrica_criterios.html

   Archivo: lista_cotejo_criterios.html
   Tipo: Checklist de verificación Sí/No

   Color: Verde (#1abc9c)

   Archivo: portafolio_evidencias.html
   Tipo: Evaluación de colección de trabajos

   Color: Rosa (#f5576c)

4. ✅ REGISTRO ANECDÓTICO
   Archivo: registro_anecdotico.html
   Tipo: Anotación de comportamientos significativos
   Uso: Registra contexto, comportamiento observado + 5 criterios
   Campos extra: Hora, contexto, comportamiento detallado
   Color: Cyan (#00f2fe)

5. ✅ ESCALA VALORATIVA
   Archivo: escala_valorativa.html
   Tipo: Evaluación gradual con colores por nivel
   Uso: 5 criterios con representación visual de escala
   Especial: Código de colores para cada nivel
   Color: Verde brillante (#38f9d7)

6. ✅ FICHA DE OBSERVACIÓN
   Archivo: ficha_observacion.html
   Tipo: Sistematización de observaciones en clase

   Color: Naranja-Rosa (#fa709a)

7. ✅ EVALUACIÓN DIAGNÓSTICA
   Archivo: evaluacion_diagnostica.html
   Tipo: Valoración inicial de competencias
   Uso: Se utiliza en la Unidad Diagnóstica (inicio del período)
   Función: Identificar nivel de partida del estudiante
   Color: Pastel (#a8edea y #fed6e3)

8. ✅ FICHA DE ANÁLISIS LITERARIO
   Archivo: ficha_analisis_literario.html
   Tipo: Evaluación específica para análisis de textos
   Uso: 5 aspectos del análisis literario
   Aspectos: Comprensión | Personajes | Recursos | Contexto | Crítica
   Color: Púrpura oscuro (#8e2de2)

🚀 CÓMO USAR EL SISTEMA COMPLETO:
═════════════════════════════════════════════════════════════════════════════

PASO 1: Abre en navegador
   file:///Users/osmer/Downloads/Sistemita/Sistemita_Nuevo/instrumentos/registro_calificaciones_v2.html

PASO 2: Completa información superior
   - Estudiante: nombre
   - Período: Bimestre I, II, etc.
   - Grado: 5to A, 6to B, etc.

PASO 3: Para CADA COMPETENCIA, elige:

   OPCIÓN A - Criterios Directos:
   ├─ Completa los 5 radios directamente (C/B/A/AD)
   └─ El sistema calcula automáticamente

   OPCIÓN B - Usar un Instrumento:
   ├─ Selecciona instrumento del dropdown
   ├─ Clic "Usar Instrumento"
   ├─ Se abre modal (ventana flotante)
   ├─ Completa el instrumento
   ├─ Clic "Guardar Respuesta"
   ├─ Modal cierra y retorna al Registro
   └─ El instrumento aparece en lista con su calificativo

PASO 4: MÚLTIPLES INSTRUMENTOS POR COMPETENCIA
   ├─ Puedes agregar Rúbrica + Lista Cotejo + Ficha Obs a LA MISMA competencia
   ├─ La Rúbrica DOMINA (decide el calificativo final)
   ├─ Los demás complementan (máximo llegan a A)
   └─ Ejemplo: Rúbrica=AD, Lista Cotejo=A → Final=AD

PASO 5: Guardar todo
   └─ Clic [💾 Guardar] → Descarga JSON completo con todo

🔑 CARACTERÍSTICAS CLAVE DEL SISTEMA:
═════════════════════════════════════════════════════════════════════════════

✓ AUTONOMÍA TOTAL
  Cada instrumento abre NUEVO sin memoria anterior
  Cada uso es independiente y limpio
  No hay contaminación de datos entre sesiones

✓ LÓGICA MINEDU IMPLEMENTADA
  5 criterios por instrumento: C (Inicio) | B (Proceso) | A (Esperado) | AD (Destacado)
  Cálculos automáticos basados en conteo:
  ├─ 5 en A = A (100%)
  ├─ 4 en A + 1 en AD = AD (>100%)
  ├─ Máx 3 en A = B
  └─ 2 o menos en A = C

✓ RÚBRICA DOMINA
  Si hay Rúbrica, su calificativo es el final
  Los demás instrumentos máximo alcanzan A (100%)
  Permite que un instrumento sea "principal" para la competencia

✓ MODALES FLOTANTES
  Los instrumentos abren como ventanas flotantes
  No navegan a otra página
  Puedes tener varios abiertos (aunque uno por vez)
  Cierran y retornan al Registro automáticamente

✓ EXPORTACIÓN JSON
  Cada instrumento guarda su resultado en JSON
  El Registro guarda TODO: estudiante, competencias, instrumentos, respuestas
  Preparado para migración de datos
